# BL4Builds 产品需求文档 (PRD)

## 一、产品概述

**产品名称**：BL4Builds.net
**定位**：Borderlands 4 玩家社区与工具站，聚焦 Build（技能加点 + 装备搭配）分享与查询。
**目标**：提供快速、直观、可信的 Build 参考与工具，成为玩家首选的 Borderlands 4 Build 数据与社区入口。

---

## 二、MVP 目标（第 0 阶段）

**上线时间**：1 天内
**核心目标**：快速上线一个能展示 Build 内容的站点，满足玩家“抄作业”需求。

### MVP 功能范围

1. **首页 Landing**

   * Hero 图 + 站点介绍
   * Top Builds 卡片（3-6 个）
   * 视频缩略图外链区（YouTube 预告、演示）

2. **Build 列表页**

   * 静态卡片列表（按角色/标签分类）
   * 搜索框（前端本地过滤标题/标签）

3. **Build 详情页**

   * 技能加点图/文字说明
   * 装备推荐（文字）
   * 使用方法/打法说明
   * 更新版本标注（Patch v1.0）

4. **关于/免责声明**

   * 声明粉丝站性质，与官方 Gearbox 无关
   * 邮箱/反馈通道

### MVP 技术选型

* 前端：Next.js 或 Vite + React + Tailwind
* 内容：Markdown/MDX 静态文件
* 部署：Cloudflare Pages / Vercel
* 分析：Cloudflare Analytics

### 验收标准

* 首页可访问，展示至少 3 个 Build 卡片和 3 个视频缩略图
* Build 详情页完整（技能、装备、打法）
* 站点移动端适配、基础 SEO 可用

---

## 三、迭代规划（未来阶段）

### 阶段 1（第 1-2 周）

* **技能树计算器**（前端模拟器，点技能保存结果）
* Build 详情页支持 **一键复制技能字符串**
* 增加装备掉落表（10-20 件核心装备）

### 阶段 2（第 3-4 周）

* **装备数据库**（筛选：元素、稀有度、掉落来源）
* Build 与装备自动关联
* Build 列表支持排序（热门、新增、Patch 适配）

### 阶段 3（第 2 个月）

* **用户投稿 Build**（表单提交，人工审核后生成页面）
* Build 社区互动（点赞、评论）
* 版本更新提示（Patch 1.1, 1.2 标签）

### 阶段 4（第 3 个月及以后）

* **国际化**（中/英双语）
* **高级搜索**（多条件组合）
* **个性化推荐**（根据职业/玩法推荐 Build）
* API 接口，支持与 Discord Bot/第三方工具对接

---

## 四、非功能性需求

* **性能**：Lighthouse > 90，移动端加载 < 3s
* **SEO**：sitemap.xml、robots.txt、Open Graph
* **安全**：HTTPS 默认，用户数据收集前期最少化
* **合规**：版权与商标免责声明

---

## 五、关键指标（KPI）

* **站点流量**：首月 PV ≥ 5,000
* **内容点击率**：首页 → Build 详情 CTR ≥ 30%
* **用户停留时长**：≥ 90 秒/Build 详情页
* **用户生成内容占比**：3 个月内 ≥ 20% Build 来自社区投稿

---

## 六、风险与对策

* **游戏版本更新快** → 设立 Patch 快速更新流程（先发 TL;DR，后补细节）
* **版权风险** → 全站免责声明，不使用 Gearbox 官方素材作为 LOGO
* **冷启动** → 前期由站点团队撰写 5-10 篇 Build，Reddit/Discord/YouTube 传播引流

---

✅ **总结**：MVP 聚焦在内容展示与快速可用；后续通过“工具 + 数据 + 社区”逐步扩展，最终形成 Borderlands 4 Build 的权威平台。
